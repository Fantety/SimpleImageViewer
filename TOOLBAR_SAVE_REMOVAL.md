# 移除Toolbar中的独立保存按钮

## 背景
在Task 17中，我们为每个编辑功能（resize、crop、format conversion、background setter）都添加了"保存"和"保存副本"两个选项。这使得Toolbar中的独立保存按钮变得多余。

## 问题
- Toolbar中有一个独立的"保存"按钮
- 每个编辑对话框中也有"保存"和"保存副本"按钮
- 功能重复，用户体验不一致
- 独立保存按钮的功能是"另存为"，与编辑对话框中的"保存"（覆盖）语义不同

## 解决方案
移除Toolbar中的独立保存按钮和相关功能：

### 1. 更新Toolbar组件
- 移除 `onSave` prop
- 移除保存按钮和分隔线
- 更新组件文档注释

### 2. 更新ImageViewer组件
- 移除 `handleSave` 函数
- 移除 `updateCurrentImagePath` 的引用（不再需要）
- 移除 Ctrl+S 键盘快捷键
- 移除Toolbar组件的 `onSave` prop传递

### 3. 更新键盘快捷键文档
移除 Ctrl+S 快捷键说明，现在支持的快捷键为：
- Ctrl/Cmd + O: 打开文件对话框
- Arrow Left (←): 上一张图片
- Arrow Right (→): 下一张图片

## 新的保存工作流

用户现在通过以下方式保存图片：

1. **编辑后保存**：
   - 打开任意编辑对话框（resize/crop/convert/background）
   - 进行编辑
   - 点击"保存"按钮 → 覆盖原图
   - 或点击"保存副本"按钮 → 另存为新文件

2. **无需独立保存**：
   - 所有编辑操作都在对话框中完成保存
   - 保存选项更加明确（覆盖 vs 副本）
   - 避免了用户忘记保存的问题

## 优势

1. **更清晰的用户体验**：
   - 编辑和保存在同一个对话框中完成
   - 用户明确知道是覆盖还是创建副本
   - 减少了操作步骤

2. **避免混淆**：
   - 不再有两个不同的"保存"功能
   - 语义更加统一

3. **代码简化**：
   - 移除了冗余的handleSave函数
   - 减少了组件间的prop传递
   - 移除了不必要的键盘快捷键

## 修改的文件
- `src/components/Toolbar.tsx`
- `src/components/ImageViewer.tsx`

## 向后兼容性
这是一个破坏性改动，但改进了用户体验：
- 移除了Ctrl+S快捷键
- 移除了独立的保存按钮
- 用户需要通过编辑对话框来保存图片
